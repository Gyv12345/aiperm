# AiPerm智能权限管理软件 用户手册

## 一、软件简介

### 1.1 软件概述

AiPerm智能权限管理软件是一款面向企业级应用的RBAC（基于角色的访问控制）权限管理系统。本软件采用前后端分离架构，后端基于Spring Boot 4.0 + Java 25构建，前端基于Vue 3 + TypeScript构建，为企业提供完整的用户认证、权限管理、系统监控、审计日志等基础能力，同时支持AI辅助的快速业务开发。

### 1.2 主要特点

1. **模块化架构**：按业务领域划分模块，模块间松耦合，便于扩展和维护。
2. **AI Native设计**：专为AI协作开发优化，提供开发技能包，支持代码自动生成。
3. **多认证方式**：支持用户名密码、短信验证码、邮箱、OAuth第三方登录及双因素认证。
4. **细粒度权限控制**：支持按钮级别的权限管理，满足精细化权限需求。
5. **类型安全**：前后端全程类型安全，通过自动生成API客户端消除类型隐患。
6. **企业级能力**：提供审计日志、文件存储、消息通知、定时任务等企业级功能。

## 三、安装/使用说明

### 3.1 系统要求

**服务器端要求：**

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2核 | 4核及以上 |
| 内存 | 4GB | 8GB及以上 |
| 硬盘 | 50GB | 100GB及以上 |
| 操作系统 | Linux（CentOS 7+/Ubuntu 18.04+） | Linux |
| JDK | JDK 25 | JDK 25 |
| 数据库 | MySQL 8.0+ | MySQL 9.x |
| 缓存 | Redis 6.0+ | Redis 7.x |

**客户端要求：**

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Firefox 90+、Safari 15+、Edge 90+ |
| 分辨率 | 1920×1080及以上 |

### 3.2 安装步骤

**1. 安装后端依赖环境：**

```bash
# 安装 JDK 25
# 安装 MySQL 8.0+ 并创建数据库
mysql -u root -p
CREATE DATABASE aiperm DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 安装 Redis 6.0+
```

**2. 部署后端服务：**

```bash
# 进入后端目录
cd backend

# 使用 Maven 构建项目
./mvnw clean package -DskipTests

# 运行后端服务
java -jar target/aiperm-backend.jar
```

**3. 部署前端服务：**

```bash
# 进入前端目录
cd frontend

# 安装依赖
pnpm install

# 构建生产版本
pnpm run build

# 将 dist 目录部署到 Nginx
```

**4. 配置 Nginx 反向代理：**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /path/to/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3.3 启动软件

**启动后端：**

```bash
cd backend
java -jar target/aiperm-backend.jar
```

后端服务启动后，可访问以下地址：
- API服务：http://localhost:8080
- API文档：http://localhost:8080/swagger-ui.html

**启动前端：**

```bash
cd frontend
pnpm run dev
```

前端开发服务器启动后，访问：http://localhost:5173

## 四、主要功能说明

### 4.1 功能一：用户认证与登录

系统支持多种认证方式登录管理后台。

**操作步骤：**

1. 打开浏览器，访问系统登录页面。
2. 选择登录方式（账号密码/短信验证码/邮箱/OAuth）。
3. 输入对应的认证信息。
4. 如已开启双因素认证，输入验证码完成二次验证。
5. 登录成功后进入系统首页。

**注意事项：**

- 默认管理员账号：admin，密码：admin123（首次登录后请立即修改密码）。
- 连续登录失败5次后，账号将被锁定30分钟。
- 建议启用双因素认证以提高安全性。

### 4.2 功能二：用户管理

管理员可以对系统用户进行全生命周期管理。

**操作步骤：**

1. 登录系统后，在左侧菜单点击「系统管理」→「用户管理」。
2. 查看用户列表，支持按用户名、手机号、状态等条件搜索。
3. 点击「新增」按钮创建新用户，填写用户基本信息、选择所属部门和角色。
4. 点击用户行的「编辑」按钮修改用户信息。
5. 点击「删除」按钮删除用户（逻辑删除）。
6. 点击「重置密码」为用户重置登录密码。

**注意事项：**

- 新建用户时必须分配至少一个角色。
- 用户名创建后不可修改。
- 删除用户为逻辑删除，可通过数据库恢复。

### 4.3 功能三：角色管理

通过角色管理实现权限的批量分配和管理。

**操作步骤：**

1. 在左侧菜单点击「系统管理」→「角色管理」。
2. 查看角色列表，显示角色名称、权限标识、状态等信息。
3. 点击「新增」创建新角色，填写角色名称、描述，并分配菜单权限。
4. 在权限树中勾选该角色可访问的菜单和按钮权限。
5. 点击「确定」保存角色及权限配置。
6. 在用户管理中为用户分配角色。

**注意事项：**

- 超级管理员角色不可删除和修改权限。
- 角色权限变更后，对应用户需要重新登录生效。
- 建议按部门或职能划分角色。

### 4.4 功能四：系统监控

提供系统运行状态的实时监控功能。

**操作步骤：**

1. 在左侧菜单点击「系统监控」。
2. 选择监控类型：
   - **在线用户**：查看当前在线用户列表，可强制下线。
   - **服务监控**：查看服务器CPU、内存、磁盘、JVM等运行状态。
   - **缓存监控**：查看Redis缓存使用情况，支持缓存清理。
3. 点击对应的监控项查看详细信息。

**注意事项：**

- 监控数据为实时数据，页面自动刷新。
- 强制下线操作需谨慎，下线后用户需重新登录。
- 缓存清理操作会影响系统性能，建议在低峰期操作。

### 4.5 功能五：审计日志

记录和查询系统操作日志及登录日志。

**操作步骤：**

1. 在左侧菜单点击「审计日志」。
2. 选择日志类型：
   - **操作日志**：查看用户的增删改操作记录，包含操作人、操作时间、操作内容等。
   - **登录日志**：查看用户登录/登出记录，包含登录IP、浏览器、登录时间等。
3. 支持按时间范围、操作人、操作类型等条件筛选。
4. 点击日志条目查看详细信息。

**注意事项：**

- 日志数据默认保留6个月，超期自动清理。
- 日志为只读数据，不可修改和删除。
- 管理员可导出日志用于审计。

### 4.6 功能六：文件存储管理

管理系统中的文件上传和存储。

**操作步骤：**

1. 在左侧菜单点击「文件管理」。
2. 查看已上传文件列表，包含文件名、大小、上传时间等。
3. 点击「上传」按钮选择本地文件上传。
4. 支持拖拽上传和批量上传。
5. 点击文件可预览或下载。
6. 点击「删除」移除不需要的文件。

**注意事项：**

- 单个文件上传大小限制默认为50MB。
- 支持的文件类型：图片、文档、压缩包等常见格式。
- 文件存储在阿里云OSS，确保网络连通。

### 4.7 功能七：审批流程

管理和处理工作流审批。

**操作步骤：**

1. 在左侧菜单点击「审批管理」。
2. 在「待办审批」中查看需要处理的审批任务。
3. 点击审批任务查看审批详情。
4. 选择「同意」或「拒绝」，填写审批意见。
5. 在「我发起的」中查看自己提交的审批进度。
6. 在「审批记录」中查看历史审批数据。

**注意事项：**

- 审批操作不可撤回，请确认后再提交。
- 审批流程由系统管理员配置，用户不可自行修改。
- 超时未处理的审批会自动提醒。

## 五、操作步骤

### 5.1 基本操作

**登录系统：**

1. 打开浏览器，输入系统地址。
2. 在登录页面输入用户名和密码。
3. 点击「登录」按钮进入系统。

**修改个人信息：**

1. 点击右上角用户头像。
2. 选择「个人信息」进入个人中心。
3. 修改昵称、手机号、邮箱等信息。
4. 点击「保存」提交修改。

**修改密码：**

1. 点击右上角用户头像。
2. 选择「修改密码」。
3. 输入旧密码和新密码（需确认）。
4. 点击「确定」完成修改。

### 5.2 高级操作

**数据权限配置：**

1. 进入「角色管理」，选择需要配置的角色。
2. 在「数据权限」选项卡中选择数据范围（全部/本部门/本部门及以下/自定义）。
3. 如选择自定义，勾选可访问的部门。
4. 点击「确定」保存配置。

**字典数据维护：**

1. 进入「系统管理」→「字典管理」。
2. 选择需要维护的字典类型。
3. 新增、编辑或删除字典数据项。
4. 字典数据修改后实时生效。

## 六、注意事项

### 6.1 使用注意事项

1. 请使用Chrome、Firefox、Safari或Edge等现代浏览器访问系统。
2. 建议屏幕分辨率为1920×1080及以上，以获得最佳体验。
3. 操作完成后请及时退出系统，避免账号被他人使用。
4. 重要操作（如删除数据、修改权限）请仔细确认后再执行。
5. 定期修改登录密码，密码长度不少于8位，建议包含字母、数字和特殊字符。
6. 请勿在公共网络环境下登录系统。

### 6.2 常见问题

| 问题 | 解决方案 |
|------|----------|
| 登录页面无法打开 | 检查网络连接，确认后端服务是否正常启动 |
| 登录后页面空白 | 清除浏览器缓存，重新登录 |
| 提示"无权限访问" | 联系管理员确认是否已分配相应角色和权限 |
| 文件上传失败 | 检查文件大小和格式是否符合要求 |
| 页面加载缓慢 | 检查网络状况，清除浏览器缓存 |
| 验证码未收到 | 检查手机号是否正确，确认短信服务是否正常 |
