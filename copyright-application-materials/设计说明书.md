# AiPerm智能权限管理软件 设计说明书

## 一、软件概述

### 1.1 软件简介

AiPerm智能权限管理软件是一款基于RBAC（基于角色的访问控制）模型的企业级权限管理系统。系统采用前后端分离的B/S架构，后端基于Spring Boot 4.0 + Java 25构建微服务，前端基于Vue 3 + TypeScript构建单页应用。系统以模块化设计为核心，提供完整的用户认证、权限管理、系统监控、审计日志、文件存储等企业级基础能力，并创新性地融入AI Native设计理念，支持AI辅助的快速业务开发。

### 1.2 开发背景

随着企业信息化建设的深入推进，权限管理作为企业信息系统的核心基础能力，面临着以下挑战：

1. **传统权限系统扩展性差**：传统单体架构的权限系统难以适应微服务和模块化的发展趋势。
2. **AI时代开发效率要求提升**：AI辅助开发成为趋势，需要权限基座系统能够适配AI协作开发流程。
3. **企业级安全合规要求日益严格**：数据安全法、个人信息保护法等法规对权限管理提出了更高的合规要求。
4. **多终端多场景适配需求**：企业应用场景日益复杂，需要灵活的权限控制能力。

### 1.3 设计目标

1. **模块化**：采用Spring Modulith实现模块化架构，业务模块按领域划分，松耦合高内聚。
2. **安全性**：基于Sa-Token实现完整的认证授权体系，支持多种认证方式和细粒度权限控制。
3. **类型安全**：前后端全程类型安全，通过代码生成消除接口调用的类型隐患。
4. **AI友好**：提供完整的开发技能包和代码生成工具，支持AI辅助快速开发。
5. **可扩展**：支持模块热插拔，新增业务模块无需修改核心代码。

## 四、详细设计

### 4.1 模块一：认证模块（auth）

#### 4.1.1 模块功能

认证模块负责用户的身份验证和令牌管理，支持以下认证方式：
- 用户名密码登录
- 手机短信验证码登录
- 邮箱验证码登录
- OAuth2.0第三方登录
- 双因素认证（2FA）

#### 4.1.2 模块接口

| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| 密码登录 | POST | /auth/login/password | 用户名密码登录 |
| 短信登录 | POST | /auth/login/sms | 手机验证码登录 |
| 邮箱登录 | POST | /auth/login/email | 邮箱验证码登录 |
| 退出登录 | POST | /auth/logout | 退出登录 |
| 获取验证码 | GET | /auth/captcha | 获取图形验证码 |
| 发送短信 | POST | /auth/sms/send | 发送短信验证码 |

#### 4.1.3 处理流程

```
用户输入凭据 → 验证图形验证码 → 验证用户凭据
    → 检查账号状态 → 生成Token → 检查2FA
    → 返回Token信息 → 前端存储Token → 跳转首页
```

### 4.2 模块二：系统模块（system）

#### 4.2.1 模块功能

系统模块是核心业务模块，包含用户管理、角色管理、菜单管理、部门管理、岗位管理和字典管理六个子模块。

**用户管理**：
- 用户的增删改查
- 用户角色分配
- 用户部门归属
- 密码重置
- 用户状态管理

**角色管理**：
- 角色的增删改查
- 菜单权限分配
- 数据权限配置
- 角色状态管理

**菜单管理**：
- 菜单的增删改查
- 菜单树形结构管理
- 权限标识配置
- 菜单类型（目录/菜单/按钮）

#### 4.2.2 模块接口

| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| 用户列表 | GET | /system/user | 分页查询用户列表 |
| 用户详情 | GET | /system/user/{id} | 获取用户详情 |
| 创建用户 | POST | /system/user | 创建新用户 |
| 更新用户 | PUT | /system/user | 更新用户信息 |
| 删除用户 | DELETE | /system/user/{id} | 删除用户 |
| 角色列表 | GET | /system/role | 分页查询角色列表 |
| 菜单列表 | GET | /system/menu | 查询菜单树 |
| 部门树 | GET | /system/dept/tree | 查询部门树 |

#### 4.2.3 处理流程

**权限校验流程：**

```
用户请求 → Sa-Token拦截器 → 检查登录状态(@SaCheckLogin)
    → 检查权限标识(@SaCheckPermission) → 通过/拒绝
```

**数据权限过滤流程：**

```
查询请求 → 获取当前用户角色 → 获取数据权限范围
    → 构建过滤条件 → 执行查询 → 返回过滤后的数据
```

### 4.3 模块三：监控模块（monitor）

#### 4.3.1 模块功能

监控模块提供系统运行状态的实时监控能力，包含在线用户监控、服务器监控、缓存监控和服务监控。

#### 4.3.2 模块接口

|------|------|------|------|

| 强制下线 | DELETE | /monitor/online/{token} | 强制用户下线 |
| 服务器信息 | GET | /monitor/server | 获取服务器状态 |
| 缓存信息 | GET | /monitor/cache | 获取缓存状态 |


### 4.4 模块四：审计模块（audit）

#### 4.4.1 模块功能

审计模块通过AOP切面自动记录系统操作日志和登录日志，支持日志查询和导出。

#### 4.4.2 模块接口

| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| 操作日志 | GET | /audit/operation | 分页查询操作日志 |
| 登录日志 | GET | /audit/login | 分页查询登录日志 |

#### 4.4.3 处理流程

```
操作执行 → @Log注解切面拦截 → 记录操作信息
    → 异步写入数据库 → 操作完成返回
```

## 七、算法设计

### 7.1 核心算法

**RBAC权限树构建算法：**

系统采用递归算法从数据库读取菜单数据，构建树形权限结构。算法步骤如下：
1. 查询所有有效菜单记录。
2. 以parent_id为0的记录作为根节点。
3. 递归查找每个节点的子节点。
4. 构建完整的权限树。

**数据权限过滤算法：**

根据角色的数据权限范围构建SQL过滤条件：
- 全部数据：不添加过滤条件。
- 本部门数据：过滤dept_id等于用户部门。
- 本部门及以下：使用部门树过滤。
- 自定义：使用指定的部门列表过滤。

### 7.2 算法流程

**权限校验流程：**

```
1. 从Token中获取用户ID
2. 查询用户拥有的所有角色
3. 合并所有角色的权限标识集合
4. 检查目标权限标识是否在集合中
5. 返回校验结果（通过/拒绝）
```

### 7.3 算法复杂度

- 权限树构建：O(n)，n为菜单总数
- 权限校验：O(1)，使用HashSet存储权限标识
- 数据权限过滤：O(n)，n为部门数量

- ElMessage：消息提示
- ElMessageBox：确认对话框

## 九、安全设计

### 9.1 安全机制

1. **认证机制**：基于Sa-Token的Token认证，Token存储在Redis中，支持过期自动清理。
2. **授权机制**：基于RBAC模型的权限控制，支持菜单权限和数据权限。
3. **密码安全**：使用BCrypt算法加密存储密码，防止彩虹表攻击。
4. **请求防护**：使用Spring Security防护CSRF攻击。

### 9.2 数据保护

1. **传输加密**：生产环境使用HTTPS加密传输。
2. **存储加密**：敏感字段（密码）加密存储。
3. **逻辑删除**：业务数据采用逻辑删除，保留数据可恢复性。
4. **SQL注入防护**：使用JPA参数化查询，杜绝SQL注入。

### 9.3 访问控制

1. **接口级别**：通过 `@SaCheckLogin` 和 `@SaCheckPermission` 注解控制接口访问。
2. **按钮级别**：前端通过权限指令 `v-hasPermi` 控制按钮显示。
3. **数据级别**：通过数据权限过滤控制用户可见数据范围。
4. **审计追踪**：所有敏感操作记录审计日志。
