## 修复说明 (2026-02-28)

### Bug #1：用户编辑不生效
**根本原因**: 后端 `UserService.update()` 方法没有处理 `roleIds`，导致用户角色关联未更新。

**修复内容**:
1. `UserRepository` 新增 `updateUserRoles()` 方法
2. `UserService.update()` 和 `create()` 中调用角色关联更新

### Bug #2：用户删除不实际删除
**说明**: 系统采用软删除机制（`deleted = 1`），所有查询都过滤 `deleted = 0`，功能正常。

### Bug #3：公告添加验证错误
**根本原因**: 前端使用 `noticeTitle/noticeContent/noticeType`，后端 DTO 使用 `title/content/type`。

**修复内容**:
1. 前端 API 类型定义 `NoticeVO/NoticeDTO` 字段名改为与后端一致
2. 公告管理页面中所有字段引用统一修改

### Bug #4：新建用户状态默认为停用
**根本原因**: 前端用户管理页面 `handleCreate()` 方法中 `status: 1`（停用）。

**修复内容**: 修改为 `status: 0`（正常）

### Bug #5：个人中心状态显示未使用字典
**根本原因**: 个人中心页面直接硬编码 `{{ status === 0 ? '正常' : '停用' }}`，未使用字典组件。

**修复内容**:
1. 引入 `useDict('sys_status')` 和 `DictTag` 组件
2. 替换硬编码为 `<DictTag :options="sys_status" :value="profile?.status" />`
